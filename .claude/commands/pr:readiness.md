Use the pr-readiness-reviewer agent to perform a comprehensive pre-PR review of all changes on the current branch against the main branch.

Provide a detailed report covering:
1. Code quality and best practices
2. Test coverage and correctness
3. Security concerns
4. Documentation completeness
5. Potential bugs or edge cases
6. Database migration safety (if applicable)
7. Any breaking changes

After the review is complete, summarize the key findings for the user with specific recommendations on what needs to be addressed before opening a PR.
